# Public Office Watch

한국 선출직 공직자의 공개 발언과 공식 행동을 근거 기반으로 추적하는 시민용 기록 대시보드입니다.

## 프로젝트 목표

이 프로젝트는 정치적 공격이나 평가 순위가 아니라, 시민이 공개 기록을 더 쉽게 확인하도록 돕는 기록형 대시보드를 목표로 합니다. 첫 MVP는 국회의원 목업 데이터만 사용하며, 이후 시장, 도지사, 구청장, 시의원 등 다른 선출직 공직자로 확장할 수 있는 구조를 준비합니다.

## MVP 범위

- 랜딩 페이지
- 국회의원 목록
- 의원 상세 대시보드
- 법안 목록
- 표결 목록
- 의제 태그 색인
- 근거 등급 배지
- 중립 분석 라벨
- 안전 안내문

## 데이터 소스 계획

현재 MVP는 결정적 목업 데이터만 사용합니다. 스크래핑, 실시간 외부 API 호출, 실제 정치인 이름 사용은 포함하지 않습니다. `lib/open-assembly/client.ts`는 향후 국회 공개 데이터 연동을 위한 타입 기반 어댑터 자리입니다.

향후 연동 후보:

- 국회 의안정보
- 국회 회의록
- 본회의 표결 기록
- 선거관리위원회 공개 자료
- 지방의회와 지자체 공식 자료

## 제품 및 법적 안전 원칙

이 앱은 명예훼손적 표현, 정치적 조롱, 공개 망신, 점수화, 순위화를 제공하지 않습니다. 모든 분석은 공개 기록을 기준으로 한 탐지 결과이며, 출처, 날짜, 근거 등급, 검토 상태를 함께 제시해야 합니다.

## 왜 비난성 라벨을 쓰지 않나

공직자의 공개 발언과 공식 행동 사이의 관계는 맥락, 절차, 수정안, 회의 불참 사유, 반론 등 다양한 요소가 필요합니다. 그래서 이 앱은 사람을 단정적으로 평가하는 표현 대신 `공식 기록 기준`, `근거 미확인`, `불일치 가능성`, `입장 변화 감지`, `판단 보류` 같은 중립 라벨만 사용합니다.

## 왜 AI 분석은 최종 판단이 아닌가

자동 탐지는 누락된 자료나 문맥을 완전히 이해하지 못할 수 있습니다. 따라서 화면에는 "이 분석은 공개 기록 기반 자동 탐지 결과이며 최종 판단이 아닙니다"라는 원칙을 표시하고, 맥락 확인과 반론 검토가 필요할 수 있음을 함께 안내합니다.

## 향후 로드맵

- 국회 공개 데이터 어댑터 실제 연동
- 출처별 동기화 작업과 변경 이력 관리
- 사람 검토 워크플로
- 반론 요청 및 공식 답변 상태 관리
- 지방자치단체와 지방의회 공직자 모델 확장
- 검색과 필터링 고도화
- 접근성 및 모바일 사용성 개선
